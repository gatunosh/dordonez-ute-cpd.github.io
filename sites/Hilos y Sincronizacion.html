Thread (Hilos)

Multi-threading
Los hilos se ejecutan de manera asíncrona en el mismo proceso
Los hilos comparten en el mismo espacio de memoria del proceso
-Lo cual requiere sincronización para ciertas tareas
-Semáforos, candados, mensajes, ...
La multitarea basada en procesos está a cargo del sistema operativo (y no de Java)
La multitarea basada en hilos sí está a cargo de Java (o del entorno de ejecución)
Java utiliza los hilos para permitir que el sistema sea asíncrono
Cuántos hilos podemos tener?
-Depende principalmente de...
-El sistema operativo y memoria
-Cuántos hilos de hardware tenemos
-Si los hilos de software realizan tareas:Intensas en uso del CPU, Intensas en I/O
Sistema operativo y memoria
-Los hilos necesitan memoria: stack
-Con un default de 1 Mb stack por thread (java de 64 bits)
-Y una RAM de 2 Gb para programas
-No podrá tener más de 2000 hilos +/-
Hilos de hardware y uso del CPU
-Ideal un hilo de software por hilo de hardware
-Dependerá realmente del uso del CPU que haga cada hilo
-hilos_hardware / carga_uso_cpu
-Donde carga_uso_cpu entre:
1 si se usa a full, CPU intensive
-0 si no se usa, I/O intensive

Hardware vs. Software threads

Hardware thread

Software thread
Un núcleo (core) físico (sin hyperthreading) - 1 hilo
-Ej>: i5 4 cores = 4 hardware threads		 Un núcleo logico (con hyperthreading) - 1 hilo
-Ej: i7 4 cores = 8 hardware threads
-Un procesador i7 tiene dos hilos por cada nucleo
